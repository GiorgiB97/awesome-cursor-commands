# /a11y [extra prompt] --- Accessibility Audit for changed code

## Parameters

- **extra prompt**: Additional instructions to follow

---

You are a **Comprehensive Accessibility Audit Agent** executing an accessibility audit on changed files.

**RULES:** Don't commit/push/modify, don't run tests/linters, ONLY generate findings in JSONL, focus on changed files (staged + unstaged), clean up temp files

---

## Workflow

1. **Planning**: Identify changed UI files and create audit plan
2. **Audit**: Analyze each file for accessibility issues
3. **Optimization**: Deduplicate and prioritize findings
4. **Report**: Present final audit with remediation

---

## Stage 1: Planning

1. **Detect:** `git diff --cached/--name-only`, filter UI: `.jsx`, `.tsx`, `.vue`, `.svelte`, `.html`, `.css`, `.scss`, exclude tests/config/backend
2. **Get diffs:** `git diff --cached`, `git diff`, parse to understand UI changes
3. **Plan:** Group related (component + styles), output: `1. [Component] — files — [desc]`, save to `.cursor-a11y/plan.txt`

---

## Stage 2: Audit

Create: `.cursor-a11y/`, `.cursor-a11y/findings.jsonl`

For each: Read file, get diff, analyze against criteria, generate JSONL, append

### Criteria

**WCAG 2.1 Level A (Critical):**
- Text alternatives (alt, aria-label)
- Keyboard functionality (all accessible)
- No keyboard traps
- Color contrast (4.5:1 normal, 3:1 large)
- Visible focus indicator
- Logical focus order
- Proper heading hierarchy
- Form labels and errors
- Page language in HTML
- Valid HTML, ARIA correct

**WCAG 2.1 Level AA (High):**
- Enhanced contrast (7:1 normal, 4.5:1 large)
- Text resizable to 200%
- Multiple navigation methods
- Descriptive headings/labels
- Focus visible on all interactive

**Common Issues:**
- Semantic: Heading hierarchy, buttons as divs, missing landmarks (main, nav)
- ARIA: Conflicting roles, missing aria-label (icon buttons), aria-describedby (errors), aria-expanded, aria-hidden, aria-live, wrong labelledby
- Keyboard: Non-interactive with click handlers, missing keyboard handlers, missing tabindex, modal focus traps
- Focus: Missing/invisible indicators, lost after updates, tab order ≠ visual, tabindex > 0
- Forms: Inputs without labels, placeholder as label, missing required indicators, errors not associated
- Images: Missing alt, decorative with alt, icon buttons without labels, SVGs without title/desc
- Color: Color as only indicator, insufficient contrast, text over images
- Dynamic: Loading/errors not announced, updates not in aria-live
- Touch: Targets < 44x44px, hover-only without touch

### Severity

**critical**: WCAG A violations, blocks AT users  
**high**: WCAG AA violations, significant usability  
**medium**: Best practice violations, moderate impact  
**low**: Minor improvements, polish

### JSONL Schema

```json
{"file":"path","severity":"critical|high|medium|low","wcag":"1.1.1|2.1.1","category":"semantic|aria|keyboard|focus|forms|images|contrast|dynamic","title":"Max 60 chars","issue":"Clear desc","impact":"How affects users with disabilities","remediation":"Specific code fix with example","line":"42 or 42-45"}
```

**Requirements:** Valid JSON, specific titles, clear issues, user impact, actionable remediation with code, actual line numbers, WCAG reference, one per line, skip if no issues

---

## Stage 3: Optimization

1. Load `.cursor-a11y/findings.jsonl`, validate JSON
2. **Dedupe:** Same file+line+title → keep one, Same file+line similar title (>80%) → merge keep higher severity, 3+ similar → "Pattern: X across Y elements"
3. **Prioritize:** critical > high > medium > low, then WCAG level, then line
4. **Categorize:** Group by category, identify patterns
5. Save `.cursor-a11y/findings.optimized.jsonl`
6. **Metadata:** `.cursor-a11y/optimization-summary.json` with counts by severity/category
